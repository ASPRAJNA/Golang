go run ‚Üí compile + execute immediately (temporary)  (mainly for dev , testing)  (
go build ‚Üí compile and keep the executable (permanent) ( deployment and production)

| Aspect             | `go run`                   | `go build`                 |
| ------------------ | -------------------------- | -------------------------- |
| Creates executable | ‚ùå Temporary                | ‚úÖ Permanent                |
| Binary visible     | ‚ùå No                       | ‚úÖ Yes                      |
| Execution          | Immediate                  | Manual (`./binary`)        |
| Speed              | Slower (compile every run) | Faster after build         |
| Use case           | Development, testing       | Production, deployment     |
| CI/CD usage        | ‚ùå Rare                     | ‚úÖ Very common              |
| Debugging          | Quick experiments          | Real execution environment |

go build ‚Üí builds a binary in the current directory  (Apps)  build here
go install ‚Üí builds and installs the binary to $GOBIN / $GOPATH/bin   (Tools)  install globally

| Feature             | `go build`        | `go install`             |
| ------------------- | ----------------- | ------------------------ |
| Creates binary      | ‚úÖ Yes             | ‚úÖ Yes                    |
| Binary location     | Current directory | `$GOBIN` / `$GOPATH/bin` |
| Global command      | ‚ùå No              | ‚úÖ Yes                    |
| Overwrites existing | Manual            | Automatic                |
| Used for apps       | ‚úÖ Yes             | ‚ö†Ô∏è Rare                  |
| Used for CLI tools  | ‚ùå No              | ‚úÖ Yes                    |
| CI / Production     | ‚úÖ Yes             | ‚ùå No                     |

go build compiles Go code and creates a binary in the current directory, mainly used for application builds.
go install compiles and installs the binary into Go‚Äôs bin directory, making it globally accessible, and is mainly used for CLI tools.

go mod tidy

cleans and synchronizes your module dependencies

Adding missing dependencies
Removing unused dependencies
Updating go.mod
Updating go.sum

go.sum is a security & reproducibility file
the exact same dependency code is used every time, everywhere.
It prevents:
Supply-chain attacks
Silent dependency changes
Version mismatches
‚ùå Never delete go.sum
‚ùå Never gitignore it

go mod tidy
‚Üí scan code
‚Üí fix deps
‚Üí clean mod
‚Üí secure sum



| Aspect          | go.mod               | go.sum              |
| --------------- | -------------------- | ------------------- |
| Purpose         | Declare dependencies | Verify dependencies |
| Edited by       | Go tooling           | Go tooling          |
| Human readable  | Yes                  | No (hashes)         |
| Version control | Required             | Required            |
| Security role   | Low                  | Very high           |
